Implemented user registration and authorization, developed a prototype database to store client information, learned how to store passwords securely using cryptography, and integrated the app with our API to provide real-time exchange rates.

Let's talk about the final stage. Have you ever found yourself in a situation where an application isn't working as expected? I'm sure you've thought: I wish I could get an answer immediately. We won't hire technical support for our training project, but we can still make a difference.
Let's introduce a new feature to our application — the chat with expert consultation.

A random question simulation will be initiated upon entering the chat support menu. Our operator will provide responses based on the simulated questions. Your task is to implement the logic for generating accurate answers in response to the random questions posed by the simulation.

Objectives:
  • You need to develop the chat logic and prepare responses for the operator based on a predefined list of questions:
  1. How do I activate my card?
  2. How can I check the balance on my card?
  3. What should I do if my card has been stolen?
  4. How can I block my card?
  5. What should I do if I lose my card?
  6. How do I order a new card?
  7. How can I update my personal information in my account?
  8. What should I do if I forget my card's password?
  9. How can I withdraw cash using my card?
  10. How can I transfer money from one card to another?
  11. How can I check the recent transactions on my card?
  12. What should I do if I receive an incorrect card statement?
  13. How can I set transaction limits on my card?
  14. How can I change my card's PIN code?
  15. How can I set up transaction notifications for my card?

  • Implement the last menu — Chat Support and create sub-items for this menu:
    • 1. Send Message, 2. Back: Choosing 1 initiates the simulation and randomly selects a question, followed by an operator's response.
    • A chat question should start like this: [Simulation] some question
  • Implement logic for operator responses:
    • After starting a chat session, append the time to every message, for example: Some chat message... dd-MM-yyyy HH:mm:ss
    • Before the operator joins the chat, show: System message: Operator NAME is connected, NAME is the operator's name.
    • An operator's response looks like this: NAME: some text
    • After the operator's response, display the message System message: Did you get an answer to your question? (Y/N). If the answer is Y, the chat ends, and the client is redirected to the main menu. If the answer is N, simulate two possible options:
      • First case: Show a message and then redirect to the main menu: System message: We're sorry, but no operators are currently available. Please contact us later.
      • Second case: Show a message and ask if the user wants to ask another question: System message: Ask another question? (Y/N). If Y, restart the random question simulation; if N, redirect to the main menu.
      • If the user's response doesn't match the suggested options, display: Invalid input
      • After the chat ends, display the message: System message: Chat completed. Redirecting to the main menu.
  • Avoid repeating simulation questions. If all questions have been asked, display: System message: Simulation completed dd-MM-yyyy HH:mm:ss. After this, redirect the user to the menu; new chat attempts are unavailable.

Hint: Responses don't need to be lengthy. For instance, the answer to the first question could be Activation possible via phone or at the nearest bank branch.

Hint: Utilize the regular expressions to identify matches in simulation questions. The rule of three can be helpful. For example, how can I change my card's PIN code? use (change, card, PIN).

There are simple response requirements that you need to consider:

  • The operator must provide a unique response for the client;
  • A response should have a length between 40 and 140 characters;
  • The response must be in English;
  • The response should not be entirely in uppercase;
  • The following chars can be used: ! ? , . - ) ( " '

For testing purposes, create a file in the main project folder called userData.txt; your database must store and read data from the specified file.
